## 3. 공통 UI 규칙

### 3.1 내비게이션
- 하단 탭 바 4개 항목: 홈 / 투표소 / 공약비교 / 모의투표
- 현재 활성 탭은 브랜드 컬러(#1D9E75)로 강조
- 모든 화면에서 하단 탭 바 고정 표시

### 3.2 컬러 시스템

| 용도 | 색상 | 비고 |
|---|---|---|
| 브랜드 메인 | #1D9E75 | 버튼, 활성 탭, 링크 |
| 브랜드 라이트 | #E1F5EE | 배경 강조, 안내 박스 |
| 브랜드 미디엄 | #5DCAA5 | 완료 스텝 표시 |
| 사전투표 강조 | #FAEEDA / #633806 | 사전투표 대상 배지 및 알림 |
| 텍스트 기본 | var(--color-text-primary) | 라이트모드 전용 |
| 텍스트 보조 | var(--color-text-secondary) | 라벨, 설명 |
| 배경 기본 | var(--color-background-primary) | 카드, 탭바 |
| 배경 보조 | var(--color-background-secondary) | 화면 전체 배경 |

### 3.3 타이포그래피
- 화면 제목: 12px / weight 500
- 섹션 레이블: 8px / 대문자 / 보조 컬러
- 본문: 10~11px / weight 400
- 보조 설명: 8~9px / 보조 컬러
- 최소 폰트 사이즈: 8px (와이어프레임 기준, 실제 구현 시 11px 이상 권장)

### 3.4 컴포넌트 공통 규칙
- 카드: 흰 배경 + 0.5px 테두리 + border-radius 8px
- 버튼(기본): 브랜드 메인 배경 + 흰 텍스트 + border-radius 8px
- 버튼(아웃라인): 투명 배경 + 브랜드 컬러 테두리 + 브랜드 컬러 텍스트
- 안내 박스: 브랜드 라이트(#E1F5EE) 배경 + 아이콘 + 9px 텍스트
- 배지: 소형 pill 형태, 용도별 색상 구분
- 아이콘은 lucide-react 사용

## 4. 화면별 상세 명세

---

### S-01. 메인 화면 (D-day 배너)

**목적:** 앱 진입 시 선거 일정과 핵심 기능으로의 접근을 제공한다.

**레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 앱 로고 "한표투표" + 서브타이틀 |
| D-day 배너 | 브랜드 메인 컬러 배경 / 선거명 + D-{N} 숫자 + 선거일 표시 |
| 빠른 메뉴 | "내 투표소" / "공약 비교" / "모의투표" 카드 3개 (우측 꺽쇠 아이콘) |
| 사전투표 일정 | 사전투표 기간 D-day 배지 + 날짜 안내 카드 |
| 하단 탭 바 | 홈(활성) / 투표소 / 공약비교 / 모의투표 |

**주요 인터랙션**
- D-day 숫자는 선거 일정 API 실시간 반영
- 빠른 메뉴 카드 탭 시 해당 화면으로 이동

---

### S-02. 선거구 설정

**목적:** 주민등록상 주소를 입력받아 선거구를 설정하고, 사전투표 대상 여부를 판별한다.

**레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 뒤로가기 + "내 선거구 설정" 타이틀 |
| 주소 입력 카드 | 시/도 드롭다운 → 구/시/군 드롭다운 (2단계 순차 선택) |
| 거주지 확인 | 체크박스: "현재 거주지가 주민등록 주소와 달라요" |
| 사전투표 안내 박스 | 체크 시 표시 / 브랜드 라이트 배경 + 안내 문구 |
| 설정 완료 버튼 | "설정 완료 · 투표소 찾기" (브랜드 메인 버튼) |
| 설정된 선거구 | 선택한 주소 + 사전투표 대상 배지 표시 |
| 하단 탭 바 | 홈 / 투표소(활성) / 공약비교 / 모의투표 |

**주요 인터랙션**
- 시/도 선택 시 구/시/군 목록 동적 갱신
- 체크박스 토글 시 사전투표 안내 박스 즉시 표시/숨김
- 설정 완료 탭 시 S-03으로 이동

**상태 변화**

| 상태 | 표시 내용 |
|---|---|
| 기본 | 드롭다운 미선택 상태 |
| 시/도 선택 후 | 구/시/군 드롭다운 활성화 |
| 거주지 체크 시 | 사전투표 안내 박스 노출 + 배지 표시 |

---

### S-03. 투표소 안내

**목적:** 설정된 주소 기준으로 주변 투표소 위치와 운영 정보를 제공한다.

**레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 뒤로가기 + "주변 투표소" 타이틀 |
| 선거구 배지 행 | 설정된 주소 텍스트 + 사전투표 대상 배지(해당 시) |
| 지도 / 목록 탭 | "지도" 탭(기본 활성) / "목록" 탭 전환 |
| 지도 영역 | 지도 플레이스홀더 (카카오맵 API 연동) |
| 투표소 목록 | 투표소명 + 주소 + 운영시간 리스트 (아이콘 포함) |
| 하단 탭 바 | 홈 / 투표소(활성) / 공약비교 / 모의투표 |

**주요 인터랙션**
- 지도 탭: 지도 위 핀으로 투표소 위치 표시
- 목록 탭: 카드형 리스트 표시
- 목록 항목 탭 시 해당 투표소 핀 지도 중앙 이동

---

### S-04. 공약 비교 — 후보자 탭

**목적:** 내 선거구 후보자의 공약을 관심 분야별로 필터링하여 카드 형태로 비교한다.

**레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 뒤로가기 + "공약 비교" 타이틀 |
| 탭 바 | 후보자(활성) / 정당(비례) |
| 관심 분야 필터 | 태그 pill: 경제 / 주거 / 환경 / 교육 / 복지 (다중 선택) |
| 섹션 레이블 | 선거구명 + 선택된 분야명 표시 |
| 후보자 공약 카드 | 기호 순서대로 나란히 배치 / 아바타 + 기호 + 소속 정당 + 공약 줄 + 원문 보기 링크 |
| 스크롤 안내 | "좌우로 스크롤하여 더 보기" 힌트 텍스트 |
| 하단 탭 바 | 홈 / 투표소 / 공약비교(활성) / 모의투표 |

**주요 인터랙션**
- 태그 탭 시 해당 분야 공약으로 필터링 (선택 태그 브랜드 컬러 강조)
- 카드 가로 스크롤로 후보자 추가 확인
- "원문 보기" 탭 시 외부 링크 이동

---

### S-05. 공약 비교 — 정당 탭

**목적:** 지방선거 시 비례대표 정당별 정책을 관심 분야별로 비교한다.

**레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 뒤로가기 + "공약 비교" 타이틀 |
| 탭 바 | 후보자 / 정당(비례)(활성) |
| 관심 분야 필터 | S-04와 동일한 태그 pill 필터 |
| 정당 정책 카드 | 기호 순서대로 / 정당 아이콘(색상 구분) + 기호 + 정당명 + 정책 줄 + 원문 보기 링크 |
| 스크롤 안내 | "좌우로 스크롤하여 더 보기" 힌트 |
| 하단 탭 바 | 홈 / 투표소 / 공약비교(활성) / 모의투표 |

**조건부 표시**
- 지방선거 기간에만 "정당(비례)" 탭 활성화
- 비지방선거 시 탭 비활성화 + "해당 선거에서는 제공되지 않습니다" 안내

### S-06. 모의투표

**목적:** 실제 투표용지와 동일한 구성으로 투표용지 1장씩 순서대로 모의투표를 진행한다.

#### 스텝 구조 (지방선거 기준 7단계)

| 스텝 | 투표용지 | 배경 색상 | 연결 공약 페이지 |
|---|---|---|---|
| 1 / 7 | 교육감 선거 | #E1F5EE (초록) | 후보자 공약 비교 (교육감) |
| 2 / 7 | 광역자치단체장 선거 | #E1F5EE (초록) | 후보자 공약 비교 (광역단체장) |
| 3 / 7 | 기초자치단체장 선거 | #FAECE7 (피치) | 후보자 공약 비교 (기초단체장) |
| 4 / 7 | 지역구 광역의원 선거 | #E1F5EE (초록) | 후보자 공약 비교 (광역의원) |
| 5 / 7 | 비례대표 광역의원 선거 | #E6F1FB (파랑) | 정당 정책 비교 (비례 광역) |
| 6 / 7 | 지역구 기초의원 선거 | #E1F5EE (초록) | 후보자 공약 비교 (기초의원) |
| 7 / 7 | 비례대표 기초의원 선거 | #E6F1FB (파랑) | 정당 정책 비교 (비례 기초) |

**각 스텝 레이아웃 구성 (상→하)**

| 영역 | 내용 |
|---|---|
| 상단 헤더 | 뒤로가기 + "모의투표" + "N / 7" 카운터 |
| 진행 스텝바 | 7칸 / 완료: #5DCAA5 / 현재: #1D9E75 / 미완료: 회색 |
| 스텝 레이블 | 현재 투표 종류 안내 텍스트 |
| 투표용지 | 실제 투표용지 형태 / 기호·정당·이름·선택 라디오 버튼 구성 |
| 공약 비교 링크 | 해당 선거 종류의 공약 비교 페이지 이동 링크 카드 |
| 이전 / 다음 버튼 | 이전(아웃라인) + 다음 투표용지(브랜드 메인) |
| 힌트 텍스트 | "결과는 저장되지 않아요" |
| 하단 탭 바 | 홈 / 투표소 / 공약비교 / 모의투표(활성) |

**완료 화면 레이아웃**

| 영역 | 내용 |
|---|---|
| 완료 아이콘 + 문구 | 체크 아이콘 + "모의투표 완료!" |
| 공약 요약 카드 | 선택한 후보/정당별 주요 공약 요약 (선거 종류별 색상 카드) |
| 하단 버튼 | "다시 해보기" (아웃라인) + "공약 비교로 이동" (브랜드 메인) |

**주요 인터랙션**
- 라디오 버튼 단일 선택
- 선택 전 "다음" 버튼 비활성화, 선택 후 활성화
- 공약 비교 링크 탭 시 S-04/S-05로 이동 (현재 선택 상태 유지)
- 모의투표 결과는 기기 외부로 저장·전송하지 않음

## 6. 디자인 주요 원칙

**중립성 원칙**
후보자와 정당은 기호 번호 순서로만 나열하며, 특정 후보·정당을 시각적으로 우선 노출하지 않는다.

**접근성 원칙**
최소 터치 영역 44×44px 준수, 라이트모드 전용(`color-scheme: light`로 시스템 다크모드 무시), 아이콘에 aria-hidden 적용.

**사전투표 대상 구분**
사전투표 대상자에게는 주황 계열(#FAEEDA / #633806) 배지와 아이콘을 일관되게 사용하여 일반 유권자 안내(초록 계열)와 시각적으로 명확히 구분한다.
