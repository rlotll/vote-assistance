## 2. 프로젝트 개요

### 2.1 배경 및 목적
20~30대 대한민국 유권자의 선거 참여율이 다른 연령대에 비해 상대적으로 낮은 문제를 해결하기 위해, 후보자 공약을 쉽게 비교하고 투표를 독려하는 웹 앱을 개발한다.

### 2.2 목표 사용자
- **주 타겟**: 대한민국 20~30대 유권자
- **특성**: 정치 관심은 있으나 정보 접근이 불편하고, 짧고 직관적인 정보 소비를 선호함

### 2.3 외부 데이터 연동
- **국가선거정보 개방포털시스템 Open API**
  - 후보자 정보 API
  - 정책·공약 API
  - 투표소 정보 API
  - 선거 일정 API
  - 선거구 API
- Kakao Maps API
  - 투표소 위치 API

### 3.2 내 선거구 설정

| ID | 요구사항 | 우선순위 |
|---|---|---|
| F-03 | 주민등록상 주소를 시/도 → 구/시/군 순서의 2단계 드롭다운으로 선택할 수 있다 | 상 |
| F-04 | 시/도 선택 시 해당 시/도의 구/시/군 목록이 동적으로 갱신된다 | 상 |
| F-05 | 현재 거주지와 주민등록 주소가 다른 경우 체크할 수 있는 옵션을 제공한다 | 상 |
| F-06 | F-05 체크 시 "사전투표 대상입니다. 전국 어디서나 사전투표소에서 투표하실 수 있어요!" 안내 메시지를 표시한다 | 상 |
| F-07 | 선택한 주소를 기준으로 국가선거정보 개방포털 API를 통해 주변 투표소 목록을 불러온다 | 상 |
| F-08 | 투표소 목록은 지도와 리스트 형태로 모두 제공하며, 투표소명·주소·운영시간을 표시한다 | 상 |

### 3.3 후보자 & 정당 공약 비교

| ID | 요구사항 | 우선순위 |
|---|---|---|

| F-10 | 후보자 탭에서 내 선거구 기준 후보자 목록을 API로 자동 로드한다 | 상 |
| F-11 | 경제, 주거, 환경, 교육, 복지 등 관심 분야 태그를 선택하면 해당 분야의 공약만 필터링된다 | 상 |
| F-12 | 후보자별 공약을 카드 형태로 나란히 비교할 수 있다 | 상 |
| F-13 | 정당 탭은 지방선거 시기에 활성화되며, 비례대표 정당별 정책을 카드 형태로 비교할 수 있다 | 상 |
| F-14 | 정당 탭에도 후보자 탭과 동일한 관심 분야 태그 필터를 적용한다 | 중 |
| F-15 | 공약 및 정책 카드에서 원문 출처 링크를 제공한다 | 중 |

### 3.4 모의투표

| ID | 요구사항 | 우선순위 |
|---|---|---|
| F-16 | 내 선거구 기준으로 실제 투표 용지와 동일한 구성의 모의 투표 화면을 제공한다 | 상 |

## 4. 비기능 요구사항

| ID | 요구사항 | 우선순위 |
|---|---|---|
| NF-01 | 앱 디자인은 모던/미니멀 스타일을 따르며, 20~30대 친화적 UI를 적용한다 | 상 |
| NF-02 | 국가선거정보 개방포털 API 응답 실패 시 오류 메시지를 표시하고 재시도 옵션을 제공한다 | 상 |
| NF-03 | 개인정보(주소, 알림 설정)는 기기 내에만 저장하며 서버에 전송하지 않는다 | 상 |
| NF-04 | iOS 및 Android 플랫폼을 모두 지원한다 | 상 |
| NF-05 | 선거 관련 정보는 중립적으로 표시하며 특정 후보/정당을 우선 노출하지 않는다 | 상 |
| NF-06 | 공약 비교 화면의 초기 로딩 시간은 3초 이내를 목표로 한다 | 중 |

## 7. 제약 사항

- 국가선거정보 개방포털 API의 데이터는 선거 기간에만 갱신되므로, 비선거 기간에는 일부 기능이 제한될 수 있다.
- 모의투표 결과는 어떠한 형태로도 저장·집계·외부 전송하지 않는다.
- 특정 후보자나 정당에 유리한 방향으로 정보를 편집하거나 배열하지 않는다.
- 선거구 설정이 완료되지 않은 상태로 다른 기능에 접근할 때, 선거구 설정을 먼저 하도록 안내한다.
